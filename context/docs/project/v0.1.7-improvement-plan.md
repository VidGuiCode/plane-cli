## 1. API Style Modernization (Priority 1)

### Problem
- Plane Cloud and newer self-hosted versions use `work-items` endpoint
- Current code defaults to legacy `issues` endpoint
- Old self-hosted instances (like the author's) still use `issues`

### Solution
Flip detection logic: try `work-items` first, fall back to `issues`

### Changes

| File | Change |
|------|--------|
| `src/commands/login.ts` | Update `detectApiStyle()` - try `work-items` first, catch 404 → try `issues` |
| `src/commands/login.ts` | Add `--api-style <issues\|work-items>` flag for explicit control |
| `src/core/config-store.ts` | Keep env var default as `issues` (backward compat) for now |

### Detection Logic
```typescript
async function detectApiStyle(client, workspaceSlug, projectId) {
  // Try modern API first
  try {
    await client.get(`.../work-items/?per_page=1`);
    return "work-items";
  } catch (err) {
    // If 404, try legacy
    if (err.status === 404) {
      await client.get(`.../issues/?per_page=1`);
      return "issues";
    }
    // Other errors, default to modern
    return "work-items";
  }
}
```

### User Impact
- New users on modern Plane: Works out of the box
- Old users (like author): Auto-detects or use `--api-style issues`
- CI/users with env vars: `PLANE_API_STYLE=issues` still works

## 4. Configuration Improvements (Priority 4)

### `PLANE_CONFIG` env var
Allow overriding config file path:
```typescript
// src/core/config-store.ts
export function getConfigPath(): string {
  return process.env.PLANE_CONFIG || path.join(getConfigDir(), "config.json");
}
```

### Retry Logic & Rate Limiting
Add to `api-client.ts`:
- Exponential backoff on 5xx errors
- Respect `Retry-After` header on 429 responses
- Configurable retry count (default: 3)

## Version Decision

**0.1.7** - Stay in 0.1.x line (current is 0.1.6)

This release is non-breaking:
- API style detection is **improved** with fallback to `issues`
- New commands are **additive** only
- New flags are **optional**
- All existing behavior preserved

**When to bump to 0.2.0:**
- Remove `issues` support entirely (breaking change)
- Change existing command signatures (breaking change)
- Drop Node.js version support (breaking change)
- Major CLI redesign or product pivot
