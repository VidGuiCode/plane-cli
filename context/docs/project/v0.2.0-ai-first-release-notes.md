# v0.2.0 AI-First Release Notes

## Goal

Make `plane-cli` easy for a cold-start AI to discover and use safely with minimal tokens.

## Public-facing changes

- Added `plane discover` as the canonical AI-first discovery surface
- Added `plane discover context`, `projects`, `issue-inputs`, `states`, `members`, `labels`, `cycles`, and `modules`
- Added normalized `plane where --json` and `plane profile --json`
- Added global `--dry-run`, `--no-interactive`, and `--compact`
- Added `--fields` support for issue list/get to reduce JSON size
- Added structured JSON error output for machine recovery
- Expanded dry-run coverage across mutating commands
- Promoted `discover` in CLI help and added an AI start hint on bare `plane`

## Recommended AI workflow

1. `plane discover context`
2. `plane discover issue-inputs --project <identifier-or-name>`
3. `plane ... --dry-run --json`
4. `plane ... --json`

## Documentation contract

- Prefer `plane discover` for new automation
- Use `where` and `profile` as compatibility views
- Use `--compact` when token budget matters
- Treat `--dry-run` output as the canonical preview of a mutation
